# v1.2 M4 — Setup steps (manual)

## 1. Create the Content Rewards PostHog project

PRD §S5 / 2026-05-05 sync: Content Rewards runs on its own PostHog account, separate from any Fanvue-wide account. **No cross-product identify.**

1. Go to https://us.posthog.com (or the EU region if preferred — pick one and stick to it).
2. Create a new Project named **Content Rewards**.
3. Note the project's Public API key (frontend) and the Private API key (backend).

## 2. Set env vars

**Vercel (frontend):**
```
VITE_POSTHOG_KEY=phc_xxxxxxxx     # public key
VITE_POSTHOG_HOST=https://us.i.posthog.com
```
Set on Production + Preview + Development. Re-deploy to pick up.

**Render (backend):**
```
POSTHOG_KEY=phc_xxxxxxxx           # same public key — posthog-node uses the public key
POSTHOG_HOST=https://us.i.posthog.com
```
Set on the `content-rewards-fanvue` web service (Settings → Environment). Render auto-restarts.

When unset, every event no-ops; code still runs.

## 3. Smoke-test events flow

After deploy:

1. **Frontend identify** — log in. Check PostHog → Persons → confirm a row appears for your `users.id` with `role` and `handle` properties.
2. **`campaign_created`** — publish a draft campaign as a creator. Confirm event in PostHog Live Events with `payout_type`, `total_budget_cents`, `is_private`.
3. **`subscriber_campaign_created`** — same as above for a per-sub campaign. Should fire alongside `campaign_created`.
4. **`submission_created`** — apply to a per-sub campaign as a clipper. Confirm event with `payout_type`, `application_mode`, `auto_approved`.
5. **`tracking_link_minted`** — auto-mode apply mints the link. Should fire after `submission_created`. Manual-mode apply fires on creator approval.
6. **`tracking_link_copied`** — clipper copies the link from the workspace. Frontend event.
7. **`tracking_link_revoked`** — creator clicks Revoke on a roster row. Confirm.
8. **`submission_marked_paid`** — creator marks a Ready row paid. Confirm.

## 4. Build the dashboards (M4.2)

In PostHog → Dashboards → New Dashboard ("Content Rewards v1.2 Funnel"). Create the following Insights:

### a. Subscriber Campaign Funnel
- Type: **Funnel**
- Steps:
  1. `subscriber_campaign_created`
  2. `tracking_link_minted` (filter: same campaign_id)
  3. `tracking_link_copied`
  4. `submission_marked_paid`
- Time window: 30 days
- Breakdown: by `campaign_id` (top 20)

### b. Click → Acquired Sub Conversion
- **Deferred** until `subscriber_attribution_synced` event lands in the M4.1.x follow-up. The event's properties will include `click_delta` and `sub_delta`.

### c. Mark-Paid → Confirm Conversion
- **Deferred** until `payment_confirmed` lands in the M4.1.x follow-up.

### d. Dispute Rate
- **Deferred** until `payment_disputed` + `dispute_resolved` land.

### e. Payment Completion by Method
- **Deferred** until `payment_confirmed` lands; properties will include `method`.

In short: dashboards a is buildable now; b–e wait for the follow-up M4.1.x event PR.

## 5. Verify identify on login + reset on logout

PostHog → Activity. Log in as user A → events should attribute to A's `users.id`. Log out → log in as user B → events attribute to B. Logout calls `posthog.reset()` so user A and B don't share a session.
