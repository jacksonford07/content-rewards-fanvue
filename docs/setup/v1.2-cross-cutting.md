## 3. Domain decision

Currently `content-rewards.fanvue.call.app` (a Vercel preview-style URL). Two options recommended:

| Option | Pros | Cons |
|---|---|---|
| **`rewards.fanvue.com` subdomain** (recommended) | Brand alignment; users see "this is a Fanvue thing"; no separate domain registration | Requires Cloudflare / Fanvue DNS team to add a CNAME |
| **`contentrewards.app` standalone** | Independent; SEO room | Pays brand-equity cost; users may not trust |

**To apply the recommended option:**
1. Add the subdomain DNS record (CNAME → `cname.vercel-dns.com`) at the Fanvue DNS owner.
2. Vercel dashboard → project `content-rewards-fanvue` → Domains → add `rewards.fanvue.com`.
3. Update OAuth redirect URIs in Fanvue's developer portal to include the new domain.
4. Update `FRONTEND_URL` env var on the Render backend service.
5. Re-run smoke tests on the new domain.

This is admin work, no code change.
