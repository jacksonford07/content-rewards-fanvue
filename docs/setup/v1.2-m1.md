# v1.2 M1 — Setup steps (manual)

## 1. Smoke-test mode modal + step 4 platforms

After Vercel redeploys from the merged PR:

1. As a creator, click **+ New campaign** → mode-selection modal pops.
   - Confirm two cards (Clip Campaign, Subscriber Campaign) with best-for hints.
   - Click **Subscriber Campaign** → modal closes; step 1 visible.
2. Walk to step 4 (Platforms) → confirm options are **Instagram, TikTok, Reddit, X**.
3. Back-button to step 1; reopen the modal manually (refresh / new draft) and pick **Clip Campaign**.
4. Step 4 → confirm options are **TikTok + Instagram only** (no YouTube, no Reddit/X).
5. Edit an existing draft → confirm modal does **not** pop (payoutType already set).

## 2. Smoke-test hub filters

1. Visit `/` (clipper hub) → open Filters sheet.
2. **Campaign type** select → pick "Subscriber campaigns" → Apply → confirm only per-sub cards (violet badge) show.
3. **Creator trust** checkbox → pick "100% paid track record only" → Apply → confirm only campaigns from creators with a clean payout history show.
4. Confirm the active filter count chip on the Filters button reflects both filters (+1 each).

## 3. Smoke-test card visuals

- Hub cards: top-left badge reads **Clip** (sky) or **Subscriber** (violet).
- Per-sub card top-right rate reads **$X / sub** (not "/ 1K").
- Per-sub cards hide the views (eye-icon) row.
- Detail page: type badge sits above the campaign title.

## 4. Smoke-test copy refresh

- Login page hero: "growth marketplace" + body mentions both modes.
- Login feature rows mention "per 1,000 views or per acquired subscriber" + Reddit/X.
- Hub `PageHeader` description mentions both modes (creator vs clipper variant).

## 5. No infra changes

No Drizzle migration, no `render.yaml` change, no Vercel env vars, no Fanvue API config. Backend `campaigns.list()` got a new optional `payout_type` query param — existing clients unaffected.
