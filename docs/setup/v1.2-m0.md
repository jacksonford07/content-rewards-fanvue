# v1.2 M0 — Setup steps (manual)

Things only Iniaki can do for this milestone. Everything else lands in the PR.

## 1. Smoke-test breadcrumbs on prod after merge

After Vercel redeploys from the merged PR:

- Log in as a creator → visit `/creator/inbox` → confirm breadcrumb reads **My campaigns › Submission inbox** and the link works.
- Log in as a clipper → visit `/submissions` → confirm breadcrumb reads **Campaigns › My submissions** and the link works.

## 2. Smoke-test OAuth scope-upgrade resilience

Use a creator account that does **not** yet have `write:tracking_links` granted (or revoke it in Fanvue first).

1. Start a new per-subscriber campaign — fill out steps 1–4.
2. On step 5, click Publish. Expect: toast "Granting tracking-link permission…" then bounce to Fanvue OAuth.
3. Approve the scope upgrade.
4. **Expected:** land back on `/creator/campaigns/{draftId}/edit` with the draft state restored.
5. **Pre-fix behaviour:** would land on `/creator/campaigns` (lost editing context).

If step 4 fails, check `localStorage.cr_post_login_redirect` is being set before the redirect (DevTools → Application → Local Storage).
