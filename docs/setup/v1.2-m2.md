# v1.2 M2 — Setup steps (manual)

## 1. Run the migration against prod (REQUIRED before deploy)

After merging the PR, before the new code can run:

```bash
cd server
DATABASE_URL="<prod neon postgres url>" pnpm db:push
```

This applies `0006_v1_2_m2_workspace.sql`:
- `campaigns.traffic_rules` (text, nullable)
- `submissions.last_clicks` (int, default 0)

Both columns default to safe values, so existing rows are fine.

## 2. Re-apply Render Blueprint to bump cron cadence

`render.yaml` changes the `sync-subscribers` schedule from `0 */6 * * *` to `*/30 * * * *` (every 30 min). Render won't pick this up automatically — you need to re-apply the Blueprint:

1. Render dashboard → Blueprints → existing instance for this repo → **Sync**.
2. Confirm `sync-subscribers` cron service shows the new schedule (`*/30 * * * *`).
3. Make sure `CRON_SECRET` is still set on the backend web service.

If the Blueprint sync errors, the fallback is editing the cron service directly in the Render UI: change schedule from `0 */6 * * *` to `*/30 * * * *`.

## 3. Smoke-test the new cron path

After cadence bump:
- Wait ~30 min after the next cron tick.
- Check Render logs for `sync-subscribers` — should show `updated: N` for some N > 0 if there are active per-sub submissions.
- In the app, open a per-sub campaign you've promoted → workspace should show Clicks > 0 within ~30 min of someone clicking the link.

## 4. Smoke-test workspace + roster

**As a clipper (per-sub apply flow):**
1. Apply to a per-sub campaign → tick the anti-bot checkbox before Apply enables.
2. After approval, workspace shows: Tracking link + copy + Traffic rules block (if creator set rules) + 3 stat cards (Clicks / Subs / Earnings).

**As a creator (per-sub owner):**
1. Visit a per-sub campaign you own → detail page shows "Manage promoters" CTA → click → land on budget page.
2. Budget page roster shows Clicks + Subs + Earnings per promoter.
3. Tracking-link names in your Fanvue dashboard read `<campaign> · Clipper N (@handle)` after the next mint.

## 5. Smoke-test traffic rules round-trip

1. Create a new per-sub campaign → step 5 → fill the Traffic rules textarea with a few lines (newlines preserved).
2. Publish.
3. Visit the campaign as a clipper, apply, get approved → workspace shows the rules block with newlines preserved.

## 6. No Vercel env var changes

Only Render needs the Blueprint re-sync. Vercel just needs the next deploy to pick up the merged code.
