# v1.2 M3 ‚Äî Setup steps (manual)

## 1. db:push picks up the new status value automatically

`submissions.status` is a `text` column with a TypeScript-side enum constraint, not a Postgres native enum. Adding the `revoked` value requires no ALTER. `pnpm db:push` against prod will:
- See no schema-level change for the status column type.
- Migrations 0006 (M2) is the latest .sql file; no new .sql needed for M3.

Just confirm `pnpm db:push` runs clean ‚Äî it will, because there's nothing to push.

## 2. No render.yaml change

Cron schedules unchanged. Auto-revoke logic lives inside the existing `sync-subscribers` cron ‚Äî same schedule (`*/30 * * * *` from M2.3).

## 3. Smoke-test manual revoke

1. As a creator with an active per-sub campaign, open `/creator/campaigns/{id}/budget`.
2. Pick a clipper in the **Accruing** section ‚Üí click **Revoke** ‚Üí confirm in the dialog.
3. Toast: "Tracking link revoked". The clipper moves to the **Revoked** group at the bottom.
4. In your Fanvue dashboard, the corresponding tracking link should be gone (or 404 if you check it directly).
5. The clipper's `pendingEarnings` value is unchanged ‚Äî you can still **Mark paid** it from the Ready section if it had progressed there. Revoking from Accruing means the earnings sit at the snapshot value.

## 4. Smoke-test auto-revoke (Fanvue-side delete)

1. As a creator, manually delete a tracking link from the Fanvue dashboard (one that's tied to a current per-sub submission).
2. Wait for the next `sync-subscribers` cron tick (~30 min).
3. Check Render logs for: `Auto-revoked submission <id>: tracking link <uuid> not in Fanvue listing`.
4. In the app, the submission should show **Revoked** status in both the creator's roster and the clipper's `/submissions` page.

## 5. Smoke-test ban-revoke interaction

Existing `ban()` already deletes the Fanvue link as part of the ban flow (predates v1.2). Confirm it still works after this PR:

1. Pick a clipper ‚Üí click **Ban**.
2. Tracking link gets DELETEd via the same `fanvueLinks.deleteLink` path.
3. Submission status set to **rejected** (NOT `revoked` ‚Äî ban is its own state).
4. The next `sync-subscribers` run won't auto-revoke because the row is already filtered out by status.

## 6. Revoke button placement

Revoke only shows on **Accruing** rows in the budget page roster. Not on Pending (nothing minted), not on Ready (sweep already happened), not on Settled (payout in progress). If a creator wants to revoke an active accruing link, that's the path. If a clipper has misbehaved more broadly, the **Ban** button is the heavier hammer.
